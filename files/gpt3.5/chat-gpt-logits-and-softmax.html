logits and softmax, neural networks. GO!

Logits are the raw outputs of a neural network before the application of the softmax function. They represent the unnormalized probabilities assigned by the network to each class. In other words, they are the values that the network computes for each class, without any kind of normalization or scaling. Logits are typically used as inputs to the softmax function, which converts them into a probability distribution over the classes.

The softmax function is a mathematical function that converts logits into probabilities. It takes as input a vector of real numbers (the logits) and outputs a probability distribution over the classes. The softmax function maps each element of the logits vector to a value between 0 and 1, such that the sum of the values is 1. The softmax function is often used as the final layer of a neural network, where it maps the outputs of the previous layer to a probability distribution over the possible classes.

In neural networks, the use of logits and softmax is common in classification tasks, where the goal is to assign a class label to an input. The logits are computed by the last layer of the neural network, and then passed through the softmax function to obtain the probabilities for each class. The class with the highest probability is then chosen as the predicted label for the input.

Overall, logits and softmax are important concepts in neural networks and are used in many machine learning tasks, especially in classification problems where assigning probabilities to each class is required.

is there any compelling use-case for logits over probs?

Logits and probabilities both have their own uses in different contexts, and there are some compelling use cases where logits are preferred over probabilities. Here are some examples:

  1. Gradient-based optimization: During training of a neural network, the loss function is optimized by computing the gradients of the loss with respect to the logits. This is because the gradients of the loss with respect to the logits are more stable and numerically well-behaved than those with respect to the probabilities. Therefore, using logits can improve the training stability and convergence of the neural network.

  2. Imbalanced datasets: In cases where the classes in a dataset are imbalanced, logits can provide a more accurate representation of the underlying distribution of the data. This is because probabilities can be heavily biased towards the majority class, leading to inaccurate predictions for the minority classes. Logits, on the other hand, are not biased in the same way and can provide a more balanced representation of the data.

  3. Interpretability: In some cases, it may be more interpretable to work with logits instead of probabilities. For example, in some medical applications, the logits may represent a continuous scale of severity or risk, which is more interpretable than discrete class probabilities.

Overall, logits and probabilities have different strengths and weaknesses depending on the context, and the choice between the two depends on the specific requirements of the task at hand.

If we were going to use fine-tuning by adding additional layers to a pretrained NN, should we use the probs or the logits?

When fine-tuning a pre-trained neural network by adding additional layers, it is generally recommended to use the logits as the input to the new layers, rather than the probabilities. The reason for this is that the logits contain more information about the original pre-trained model's predictions, and using logits can improve the stability and convergence of the fine-tuning process.

Using probabilities as inputs to the new layers can also lead to numerical instability, especially if the probabilities are very close to 0 or 1. This is because small changes in the probabilities can lead to large changes in the output of the softmax function, which can in turn cause the gradients to explode or vanish during training.

Therefore, it is generally better to use the logits as inputs to the new layers, and then apply the softmax function only on the final layer to obtain the probabilities. This approach ensures that the gradients are more stable during training, and that the fine-tuned model is better able to capture the nuances of the new task.

When fine-tuning a pre-trained neural network, it is generally recommended to connect the new layers to the penultimate layer (i.e., the layer before the logit layer) rather than the logit layer itself.

The reason for this is that the penultimate layer contains the high-level features that the pre-trained model has learned from the original task, which are likely to be more transferable to the new task than the logits. Connecting the new layers to the penultimate layer allows the fine-tuning process to focus on learning task-specific features that build on the pre-trained model's existing feature representations.

In contrast, connecting to the logit layer may not be as effective, since the logits are optimized for the original task and may not be directly relevant to the new task. Furthermore, the gradients with respect to the logits tend to be less stable than those with respect to the penultimate layer, which can make fine-tuning more difficult and less effective.

Overall, connecting to the penultimate layer is a good starting point for fine-tuning a pre-trained neural network, and allows for more effective transfer learning to the new task. However, as with all aspects of neural network design, it is important to experiment and fine-tune the specific architecture and hyperparameters for the particular task at hand.
