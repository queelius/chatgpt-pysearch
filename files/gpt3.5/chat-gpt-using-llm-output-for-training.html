LLMs like GPT need a lot of data to train on. How useful is the data from a previous LLM's output?

The data generated by a previous language model (LLM) like GPT can be useful for training a new model, but it's important to consider the quality and relevance of the generated data.

On one hand, using the output of a previous LLM can provide a large amount of data for training a new model, which can potentially improve the new model's performance. However, the quality of the generated data can be highly dependent on the quality of the original training data used to train the previous LLM. If the original training data was biased or contained errors, these biases and errors can be perpetuated in the generated data and affect the performance of the new model.

On the other hand, using the output of a previous LLM can also introduce the problem of "data memorization", where the new model simply memorizes the patterns and responses of the previous model without truly understanding the underlying language rules and semantics. This can limit the new model's ability to generate novel and diverse responses.

In summary, the usefulness of the data from a previous LLM's output depends on various factors, such as the quality and relevance of the generated data, the biases and errors in the original training data, and the potential for data memorization. It's important to carefully evaluate and preprocess the data before using it for training a new model.

Using the reward policy learned from reinforcement learning with human feedback (RLHF) to weigh the data generated by a previous LLM could be a promising approach, as it leverages the feedback from human evaluators to guide the weighting process.

RLHF is a technique that uses human feedback to improve the performance of an LLM by fine-tuning the model's output based on a reward signal provided by human evaluators. The reward signal can be used to adjust the weights or values assigned to the data generated by a previous LLM, based on how well the generated responses align with the desired output.

For example, if the RLHF reward policy indicates that a response generated by the previous LLM is high-quality and aligns well with the desired output, then the weight or value assigned to that response could be increased, while responses that receive a lower reward could be weighted less.

By using the RLHF reward policy to weigh the data generated by a previous LLM, the new model can learn from the feedback provided by human evaluators and focus on high-quality and relevant data, while reducing the impact of biased or low-quality data.
